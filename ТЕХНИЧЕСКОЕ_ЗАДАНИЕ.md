# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Веб-приложение "FinService" - Каталог финансовых услуг

---

## 1. ОБЩИЕ СВЕДЕНИЯ О ПРОЕКТЕ

### 1.1 Название проекта
**FinService** - Каталог финансовых услуг и продуктов

### 1.2 Тематика приложения
Веб-приложение для каталога и сравнения финансовых продуктов и услуг различных банков и страховых компаний (кредиты, вклады, кредитные карты, ипотека, страховые продукты).

### 1.3 Команда разработчиков
- Стєпа (Stepan) - Разработчик Full-Stack

### 1.4 Назначение приложения
Предоставить пользователям удобный инструмент для поиска, фильтрации и сравнения финансовых предложений различных банков и страховых компаний по категориям и географическому расположению.

## 2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 2.1 Основной функционал
1. **Авторизация и аутентификация**
   - Регистрация нового пользователя
   - Вход в систему
   - Выход из системы
   - Восстановление пароля (на перспективу)

2. **Просмотр каталога финансовых услуг**
   - Отображение списка финансовых предложений
   - Поиск по названию и описанию предложения
   - Фильтрация по категориям (кредиты, вклады, карты, ипотека, страхование)
   - Фильтрация по городам (11 крупных городов РФ)
   - Сортировка по дате добавления (новейшие/старейшие)

3. **Профиль пользователя**
   - Просмотр и редактирование личной информации
   - Изменение пароля
   - Сохранение избранных предложений (на перспективу)

4. **Навигация и UI**
   - Главная страница с приветствием
   - Шапка сайта (Header)
   - Подвал сайта (Footer)
   - Кнопка чата для общения (Chat Button)

### 2.2 Состав приложения (Страницы)

## 3. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### 3.1 Язык реализации
- **Backend**: Python 3.x (Django Framework)
- **Frontend**: JavaScript (React 18+ с Vite)
- **База данных**: SQLite

### 3.2 Архитектура приложения

#### Frontend (React + Vite)
```
Frontend (SPA - Single Page Application)
├── React Router для навигации между страницами
├── Компоненты для UI (Header, Footer, Controls, Cards)
├── Локальное хранилище (localStorage) для кэширования данных пользователя
└── SCSS для стилизации
```

#### Backend (Django REST Framework)
```
Backend (REST API)
├── Django приложение (server_cm)
├── REST API endpoints для:
│   ├── Аутентификации (регистрация, вход, выход)
│   ├── Получения списка финансовых услуг
│   ├── Фильтрации и поиска
│   └── Управления профилем пользователя
└── SQLite база данных для хранения данных
```

#### Взаимодействие
```
User Browser
    ↓ HTTPS
React Frontend (Vite)
    ↓ REST API (JSON)
Django Backend (REST Framework)
    ↓ ORM
SQLite Database
```

### 3.3 Стек технологий

**Frontend:**

- SCSS (для стилизации)

**Backend:**

## 4. СТРУКТУРА БАЗЫ ДАННЫХ

### 4.1 Таблица Users (Пользователи)
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(150) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    first_name VARCHAR(100),
    last_name VARCHAR(100),
    phone VARCHAR(20),
    city VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.2 Таблица FinancialProducts (Финансовые услуги)
```sql
CREATE TABLE financial_products (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    category VARCHAR(50) NOT NULL, 
        -- 'credit', 'deposit', 'card', 'mortgage', 'insurance'
    bank_name VARCHAR(150) NOT NULL,
    city VARCHAR(100) NOT NULL,
    conditions TEXT,
    rate_from DECIMAL(5, 2),
    rate_to DECIMAL(5, 2),
    is_featured BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.3 Таблица UserFavorites (Избранные услуги пользователя)
```sql
CREATE TABLE user_favorites (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    product_id INTEGER NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (product_id) REFERENCES financial_products(id),
    UNIQUE(user_id, product_id)
);
```

### 4.4 Таблица Cities (Города)
```sql
CREATE TABLE cities (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(20) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    region VARCHAR(100)
);
-- Предустановленные города:
-- moscow, spb, kazan, novosibirsk, ekaterinburg, 
-- krasnodar, nizhny, rostov, ufa, vladivostok
```

### 4.5 Таблица Categories (Категории)
```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    icon VARCHAR(50)
);
-- Предустановленные категории:
-- credit (Кредиты), deposit (Вклады), card (Карты), 
-- mortgage (Ипотека), insurance (Страхование)
```

### 4.6 Связи между таблицами
```
users (1) ──── (N) user_favorites ──── (1) financial_products
financial_products (N) ──── (1) categories
financial_products (N) ──── (1) cities
```

## 5. API ENDPOINTS (REST)

### 5.1 Аутентификация
```
POST   /api/auth/register          - Регистрация пользователя
POST   /api/auth/login             - Вход пользователя
POST   /api/auth/logout            - Выход пользователя
```

### 5.2 Финансовые услуги
```
GET    /api/products               - Получить список всех услуг
GET    /api/products?category=X    - Фильтр по категории
GET    /api/products?city=X        - Фильтр по городу
GET    /api/products?search=X      - Поиск по названию
GET    /api/products/:id           - Получить услугу по ID
POST   /api/products               - Создать услугу (админ)
PUT    /api/products/:id           - Обновить услугу (админ)
DELETE /api/products/:id           - Удалить услугу (админ)
```

### 5.3 Профиль пользователя
```
GET    /api/profile                - Получить данные профиля
PUT    /api/profile                - Обновить профиль
PUT    /api/profile/password       - Изменить пароль
```

### 5.4 Избранное
```
GET    /api/favorites              - Получить избранные услуги
POST   /api/favorites/:id          - Добавить в избранное
DELETE /api/favorites/:id          - Удалить из избранного
```
